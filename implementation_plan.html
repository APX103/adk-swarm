Phase 0 实施计划：主 Agent + 前端子 Agent

基于 ADK + A2A + Node/Docker，实现“一句话生成可运行的 Vite+React 前端工程”
待讨论确认 · 2026-06-16

0. 目标与范围

本 Phase 0 要验证整个“主 Agent 委派 → 远程子 Agent 生成工程 → 子 Agent 在容器内调通 → 返回工程”的闭环。

  • 主 Agent：用 Python ADK 实现，作为用户对话入口。收到“写个前端页面”类需求后，通过 A2A 把任务派发给前端子 Agent。
  • 前端子 Agent：用 Node.js 实现，跑在 Docker Container 里。擅长 Vite + React，能生成完整工程并在容器内完成 npm install + npm run build（或启动 dev server 验证）。
  • 交付物：一个可本地一键运行的 PoC，包含主 Agent、前端子 Agent、Docker 配置和说明文档。
已确认前提：不同 Agent 跑在不同 Container；外部接入强制 A2A；主 Agent 模型自主决策为主；不记录 Agent 间完整 Message，只记录 Task 级元数据。

3. 组件设计

3.1 主 Agent（main_agent/agent.py）

  • 定义一个 LlmAgent 作为 root agent。
  • 通过 RemoteA2aAgent 把前端子 Agent 注册为 sub_agent：
from google.adk.agents.remote_a2a_agent import RemoteA2aAgent, AGENT_CARD_WELL_KNOWN_PATH

frontend_agent = RemoteA2aAgent(
    name="frontend_agent",
    description=(
        "A specialist agent that builds runnable Vite + React frontend projects. "
        "Give it a description of the UI and it returns a working project archive."
    ),
    agent_card="http://localhost:8001/.well-known/agent-card.json",
    use_legacy=False,
)

root_agent = Agent(
    model="gemini-2.5-flash",
    name="main_agent",
    instruction="""
        You are a helpful assistant. When the user wants a frontend page or app,
        delegate the task to 'frontend_agent'. Summarize its result back to the user.
    """,
    sub_agents=[frontend_agent],
)

3.2 前端子 Agent（ADK TypeScript 自定义 BaseAgent）

前端子 Agent 不依赖 LLM 做路由，而是一个确定性自定义 Agent（继承 ADK TS 的 BaseAgent）。 ADK 的 toA2a() 会把它包装成标准的 A2A Server，自动处理 Agent Card 与 JSON-RPC 通信。

暴露的端点由 toA2a() 自动生成：

Path说明
/.well-known/agent-card.jsonAgent Card，供主 Agent 发现
/a2a（JSON-RPC）接收 Task，返回 Task 状态与 Artifact

内部执行流程：

  1. 解析需求：从 InvocationContext.userContent 提取用户对前端的描述。
  2. 生成工程：调用 OpenAI 兼容 API，生成 package.json、vite.config.ts、index.html、src/App.tsx、src/main.tsx 等。
  3. 写盘：在容器内 /workspace/projects/<task-id>/ 写入文件。
  4. 安装依赖：执行 npm install。
  5. 构建验证：执行 npm run build，确保无报错。
  6. dev server 验证（必须）：启动 npm run dev，请求 / 确认返回 200。
  7. 保存 Artifact：通过 FileArtifactService 把 project.tar.gz 保存到 /workspace/artifacts。
  8. 返回结果：yield 最终 Event，actions.artifactDelta 指向 project.tar.gz，文本说明构建结果。

3.3 主 Agent 的产物文件服务

Python 主 Agent 在收到前端子 Agent 返回的 artifact 后，会把它写入本地 main_agent/artifacts/<task-id>/project.tar.gz， 并启动一个 FastAPI 静态文件服务（挂载 /artifacts 路由）。 最终回复用户时，主 Agent 会给出下载链接，例如： 

http://localhost:8080/artifacts/<task-id>/project.tar.gz

3.3 LLM 生成策略

为了提升成功率，不一次性生成整个工程字符串，而是让 LLM 输出 JSON Schema，再映射到文件：

{
  "projectName": "todo-app",
  "files": {
    "package.json": "{...}",
    "vite.config.ts": "import ...",
    "src/App.tsx": "export default function App() {...}",
    ...
  }
}

也可以先让 LLM 输出文件清单，再逐个生成。具体策略可在实现中迭代。

5. Docker 设计

5.1 前端子 Agent Dockerfile

FROM node:20-slim

WORKDIR /app

# 安装系统依赖（如需）
RUN apt-get update && apt-get install -y --no-install-recommends \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*

# 复制并安装 Agent 自身依赖
COPY package*.json ./
RUN npm ci

# 复制源码
COPY . .
RUN npm run build

# 暴露 A2A 服务端口
EXPOSE 8001

CMD ["node", "dist/server.js"]

services:
  frontend_agent:
    build: ./frontend_agent
    ports:
      - "8001:8001"
    environment:
      - PORT=8001
      - OPENAI_BASE_URL=${OPENAI_BASE_URL}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - OPENAI_MODEL=${OPENAI_MODEL:-glm-4-flash}
    volumes:
      - frontend_workspace:/workspace
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8001/.well-known/agent-card.json"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  frontend_workspace:
说明：容器内需要能跑 npm，所以基础镜像用 node:20-slim。Agent 代码跑在 Node 上，生成工程时再用同一个 Node 环境执行 Vite/React 命令。

7. 开发步骤

  1. 搭建前端子 Agent：初始化 Node 项目，安装 @google/adk、openai、TypeScript、Express。
  2. 实现自定义 BaseAgent：继承 BaseAgent，在 runAsyncImpl 中解析用户需求、生成工程、构建验证、保存 artifact。
  3. 暴露 A2A 服务：用 toA2a(agent, { app, port: 8001, artifactService: new FileArtifactService('/workspace/artifacts') }) 启动服务。
  4. 实现工程生成逻辑：调用 OpenAI 兼容 API 生成 Vite+React 文件，写盘，执行 npm install / npm run build / npm run dev 验证。
  5. 实现 artifact 返回：打包为 project.tar.gz，通过 FileArtifactService 保存，并 yield 带 artifactDelta 的最终 Event。
  6. 编写 Dockerfile & docker-compose：确保 docker compose up 能拉起前端 Agent。
  7. 搭建主 Agent：Python ADK 项目，配置 RemoteA2aAgent 指向 http://localhost:8001。
  8. 实现产物文件服务：FastAPI 服务挂载 /artifacts，把从 A2A 收到的 artifact 保存并提供下载。
  9. 端到端验证：docker compose up 启动前端 Agent，python main_agent/file_server.py 启动文件服务，adk web 启动主 Agent，输入“帮我做一个 TODO 页面”，观察是否成功返回下载链接。

8. 验收标准

  • 用户通过主 Agent 发送自然语言需求，主 Agent 成功将任务派发给前端子 Agent。
  • 前端子 Agent 在 Docker 容器内生成一个可 npm run build 通过的 Vite+React 工程。
  • 主 Agent 能拿到工程产物（artifact / 下载链接 / 路径），并向用户反馈结果。
  • 整个链路可重复运行，且只需要 docker compose up + adk web 即可启动。
