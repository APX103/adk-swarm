6. eino_agent (Go) — 已实现的跨语言 Agent

核心亮点：跨语言双向 A2A 调度已完全调通

eino_agent 是一个用 Go 语言 + CloudWeGo Eino 框架编写的 双向 A2A Agent。 既是 A2A 服务端（被 main_agent 调用查天气），又是 A2A 客户端（回调 main_agent 获取时间、调度 comedian 讲笑话）。 自带 Dev Web UI（:8005/ui），可独立对话和测试。

🐍 Python 子 Agent (ADK 自动)

  • 用 to_a2a() 自动生成 A2A 服务
  • AgentCard 由 AgentCardBuilder 自动构建
  • JSON-RPC 路由由 a2a-sdk 处理
  • Agent 定义 = Agent(name, model, tools, ...)
  • 工具 = Python 函数 + @tool 装饰器

🔵 Go eino_agent (手动实现 + 双向调度)

  • net/http 手动实现 A2A Server + Client
  • AgentCard 硬编码 + SERVICE_NAME 动态 URL
  • Eino Runner (adk.NewRunner) 执行 agent
  • Agent = adk.NewChatModelAgent()
  • 3 个工具: get_weather / ask_main_agent / ask_comedian
  • Dev UI: /ui 内嵌 HTML 聊天界面
  • ~600 行 Go 代码，单文件 main.go

      "role": "user",

    "status": {
      "state": "completed",
      "message": {
        "messageId": "msg-xxx",
        "role": "agent",
        "parts": [{"kind": "text", "text": "杭州: 雷阵雨, 31°C, 湿度80%, 风力5级"}]

    }
  }
}

9. 已知性能瓶颈
瓶颈影响可优化?
LLM 调用次数 每步 3-6s，编排任务需 5+ 次调用 ✅ 并行调用 / 精简 prompt
SSE 事件过多 "你好" 产生 82 个 partial 事件，前端逐条解析 ✅ 合并 token / 前端增量渲染
Thought 事件 44 个 thought 事件无用户价值，增加解析负担 ✅ 前端过滤 / 服务端不发
HTTP 连接开销 A2A 子 Agent 调用每次新建连接 ✅ requests.Session 连接池
LLM 模型延迟 glm-4.5-air 首字 ~1s ⚠️ 换更快模型或本地模型
